# LOCALIZATION_PLAN.md — STEP 2.3 Multilingual Content Strategy

**Date**: April 2026
**Locales**: FR (French), ES (Spanish), IT (Italian), PT (Portuguese)
**English**: Source language — always produced first

---

## Principles

1. **Localization, not translation**: Every localized article must adapt examples, legal references, sources, and cultural context to the target market
2. **Local sources required**: French articles cite HEC/INSEAD/Bpifrance, Spanish cite IESE/ICO, Italian cite Bocconi/SIMEST, Portuguese cite Nova SBE/IAPMEI
3. **Local regulations**: Entity structures (SAS, GmbH, SL, SRL), tax regimes (Dutreil, régimen de emprendedores), financing programs differ by country
4. **Local examples**: Real companies, real M&A deals, real searchers from the target country when available
5. **Quality threshold**: Every localized article must read as if originally written in that language by a native-speaking ETA professional

---

| Tier | Criteria | FR | ES | IT | PT |

## Localization workflow per article

1. **Select EN article** from priority list
2. **Research local adaptation**: Identify which sections need country-specific rewrites (legal, tax, financing, sources)
3. **Draft localized version**: Not a translation — a rewrite with local context. Sections may be added/removed vs EN version
4. **Add local sources**: Minimum 2 local sources per article
5. **SEO optimization**: Local keyword in title, H2s, meta description
6. **Review**: Native speaker review for natural language quality
7. **Publish**: Add to article registry, update i18n-registry, add to sitemap
8. **Cross-link**: Link to other localized articles and EN fallbacks where localized version doesn't exist

## Technical requirements per locale launch

Before switching a locale to indexable:

- [ ] 20+ articles published and reviewed
- [ ] All articles have localized JSON-LD (Article schema in local language)
- [ ] Localized meta snippets (title, description)
- [ ] hreflang tags correctly set (EN ↔ locale)
- [ ] Sitemap includes localized URLs
- [ ] robots.txt allows locale
- [ ] i18n-registry updated: `INDEXABLE_LOCALES` includes locale
- [ ] Navigation and UI strings translated in `messages/{locale}.json`
- [ ] OG image generates correctly for locale
- [ ] No broken links within locale
- [ ] GSC submitted for locale
